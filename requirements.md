# 直播音频替身程序 - 需求文档

## 产品名称
**StreamVoice Avatar** (直播声纹替身)

## 核心功能
主播疲劳时一键切换至"录音回放模式"，播放经实时处理的近期音频，规避平台声纹检测，同时保持自然听感。

## 技术架构

### 1. 录制模块
- **格式**: MP3, 128kbps, 44.1kHz, 单声道（节省空间，直播通常单声道足够）
- **缓冲区**: 环形缓冲区，固定保存最近 30 分钟音频
- **存储**: 内存 + 临时文件双保险，防止崩溃丢失

### 2. 声纹对抗处理（核心）
**目标**: 让平台声纹识别认为是不同人，但人耳听不出异常

**技术栈**:
- **音调微调**: ±1.5 半音随机偏移（人耳察觉阈值约 ±2 半音）
- **音色微调**: 共振峰偏移（Formant Shift）±5%，改变声纹特征
- **语速微调**: 时间拉伸 ±3%，改变节奏特征
- **轻微混响**: 添加 20-50ms 早期反射，模糊声纹边界
- **微扰噪声**: -60dB 以下高斯白噪声，干扰声纹提取

**实现库**: `librosa` + `soundstretch` 或 `rubberband`

### 3. 去重逻辑（防录播检测）
- 播放位置随机偏移：在当前播放点 ±10 秒内随机选取
- 片段长度随机：3-8 秒随机截取，避免固定节奏
- 播放顺序打乱：不按时间顺序，而是随机选取非重复片段

### 4. 播放模块
- **输出设备**: 虚拟音频线路（VB-Cable 或类似）或指定声卡
- **延迟**: < 100ms 处理延迟
- **淡入淡出**: 50ms 淡入淡出，避免突兀切换

## 技术选型

- **语言**: Python 3.11
- **GUI**: PyQt6（原生感强，性能好）
- **音频**: 
  - 录制: `sounddevice` + `soundfile`
  - 处理: `librosa` + `pedalboard` (Spotify出品)
  - 播放: `sounddevice`
- **打包**: PyInstaller (单exe文件)

## 关键算法参数

```python
# 声纹对抗处理参数
VOICE_MASK_CONFIG = {
    "pitch_shift": (-1.5, 1.5),      # 半音
    "formant_shift": (-0.05, 0.05),  # 共振峰 ±5%
    "time_stretch": (0.97, 1.03),    # 时间拉伸
    "reverb_room_size": 0.15,        # 混响房间大小
    "noise_floor": -60,              # dB
}

# 去重随机参数
DEDUP_CONFIG = {
    "position_jitter": 10,           # 秒
    "min_segment": 3,                # 秒
    "max_segment": 8,                # 秒
    "repeat_threshold": 0.3,         # 30%相似度判定为重复
}
```

## 使用流程

1. 启动程序，选择输出设备（虚拟声卡）
2. 点击"开始录制"（或自动开始）
3. 正常直播，程序后台持续录制
4. 主播疲劳时，点击"开始替身"
5. 程序自动处理并播放近期录音
6. 主播休息好后，点击"恢复直播"

## 防检测策略总结

| 检测维度 | 对抗手段 |
|---------|---------|
| 声纹特征 | 共振峰偏移 + 音调微调 |
| 语速节奏 | 随机时间拉伸 |
| 音频指纹 | 微扰噪声 + 轻微混响 |
| 录播判定 | 随机片段选取 + 非顺序播放 |
| 重复内容 | 环形缓冲区 + 片段去重 |
